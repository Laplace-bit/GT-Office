# GT Office v0.1.6

GT Office `v0.1.6` is published through a tag-driven GitHub Actions workflow.

## Highlights

- Release automation now requires the GitHub Release to include a macOS `.dmg`, a Windows NSIS installer, and a Linux `.deb` package.
- File preview, terminal foundation, channel management, and installer fixes from `v0.1.5` remain included in this release line.

## Packaging and Distribution

- macOS publishes both a `.dmg` and a zipped `.app` bundle from the runner.
- Windows publishes the NSIS installer.
- Linux publishes AppImage and `.deb` bundles.

## Notes

- The repository release workflow uses the existing `npm run build:tauri` entrypoint on each host runner.
- macOS public distribution still depends on Apple signing and notarization credentials; without them, the published DMG is intended for manual testing or internal distribution.
- If a platform asset needs to be replaced after release publication, use the manual re-upload flow documented in `docs/release-process.md`.
