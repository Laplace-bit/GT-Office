# GT Office v0.1.5

GT Office `v0.1.5` is published through a tag-driven GitHub Actions workflow.

## Highlights

- File preview experience is more consistent, with unified preview tabs and PDF support.
- Terminal and workspace plumbing continue to harden the desktop foundation.
- Channel management, agent installation, and unit coverage received stability fixes.

## Packaging and Distribution

- macOS publishes a zipped `.app` bundle from the runner and a `.dmg` when signing and notarization are configured.
- Windows publishes the NSIS installer.
- Linux publishes AppImage and `.deb` bundles.

## Notes

- The repository release workflow uses the existing `npm run build:tauri` entrypoint on each host runner.
- macOS public distribution still depends on Apple signing and notarization credentials, which are not wired into the workflow yet.
- If a platform asset needs to be replaced after release publication, use the manual re-upload flow documented in `docs/release-process.md`.
