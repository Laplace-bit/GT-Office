# vbCode 上下文交接文档

## 当前里程碑与本轮目标

- 里程碑：M2 MVP 完整
- 任务：T-172
- 本轮目标：把外部通道 reply relay 从“rendered-screen 单源”升级为“session log 最终正文 + rendered-screen 预览/交互态”的双源架构，并补上 Telegram 方向键交互回流。

## 已完成

- **Session log 领域能力已落地**：
  - 新增 `crates/vb-session-log`，抽象 `SessionLogBinding / SessionLogProvider / SessionLogPollOutcome / SessionLogHealth`。
  - Claude 绑定参考 `~/.claude/projects/<project>/sessions-index.json + *.jsonl`。
  - Codex 绑定参考 `~/.codex/sessions/**/*.jsonl`，并按 `session_meta.payload.cwd` 过滤。
  - 现已具备按 `tool_kind + resolved_cwd + dispatched_text` 建立 session log 绑定，并支持增量轮询与 assistant 文本累计。
- **外部 reply session 已切成双源状态机**：
  - `ExternalReplyRelaySession` 新增 `resolved_cwd / session_log_binding / session_log_text / session_log_health`。
  - `take_external_reply_dispatch_candidates()` 现在区分：
    - preview：`rendered-screen > vt`
    - finalize：`session-log > rendered-screen > vt`
  - `ExternalReplyBodySource` 已标准化为 `session-log / rendered-screen-fallback / vt-fallback`，并映射到 `relayMode + confidence`。
  - finalize 仍以 live terminal 边界为准；只要 interaction prompt 仍在，就不会 finalize。
- **交互 prompt 升级为双模式**：
  - `ExternalInteractionPrompt` 新增 `controls / control_mode / selected_index`。
  - 语义菜单继续生成 `SubmitText` 按钮。
  - 新增纯方向型菜单提取：支持从 rendered-screen 中识别 selected option，并生成 `Up/Down/Enter/Esc/Tab` 控件。
  - 为避免交互态期间错误替换原 reply session，`bind_external_reply_session()` 已对“prompt 活跃时的控制输入”做保守吸收。
- **外部通道出站与回流已接通**：
  - `channel_sinks` 现在成为 reply / interaction 的统一出站层。
  - Telegram interaction callback 支持：
    - `gto:<submit_text>`
    - `gto-key:up|down|enter|esc|tab`
  - `process_external_inbound_message()` 会优先识别上述 callback，并直接把文本或 ANSI 控制序列写回同一 live terminal session，而不是重新走 route dispatch。
  - Feishu v1 仍只发最终文本；若存在 interaction prompt，会发送提示文本但不提供远程选择控件。
