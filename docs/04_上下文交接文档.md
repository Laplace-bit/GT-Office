# vbCode 上下文交接文档

## 当前里程碑与本轮目标

- 里程碑：M2 MVP 完整
- 任务：T-184
- 本轮目标：分析并修复 Windows 生产包启动时闪现命令行窗口的问题，补齐启动期残留的 `system git` 控制台子进程静默化

## 已完成

- 已重新梳理 Windows 闪窗根因链路：
  - Tauri 主进程在生产包中是 GUI subsystem，没有可复用的父控制台
  - 只要后台启动的是控制台子进程（`curl`、`git`、CLI 安装器等），Windows 就可能为它分配新的命令行窗口
  - 历史上 Telegram polling 每 `1.5s` 调一次 `curl`，这条路径已在 `process_utils` 中修复
  - 本轮进一步确认 `ShellRoot` 在窗口绑定已有 workspace、手动打开 workspace 后会主动执行 `git_status`
  - `git_status` 虽优先走 `git2`，但 `vb-git` 仍会在 branch/status fallback 及其他 Git 操作里走 `system git`
- 已完成代码修复：
  - `crates/vb-git/src/lib.rs`
    - 新增 `configure_background_command()`
    - 在 `run_git()` 内统一为 Windows `git` 子进程注入 `CREATE_NO_WINDOW`
    - 覆盖 `status/log/branch/checkout/fetch/pull/push/stash` 等所有复用 `run_git()` 的 system git 路径
    - 非 Windows 平台保持 no-op，不改变 Linux/macOS 行为
- 文档已同步：
  - `docs/03_项目开发进度跟踪.md`
  - `docs/04_上下文交接文档.md`
