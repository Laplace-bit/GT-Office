## 已完成

- **去掉自动安装**：
  - `apps/desktop-tauri/src-tauri/src/mcp_bridge.rs`：bridge 启动不再自动写 CLI MCP 配置。
  - `apps/desktop-tauri/src-tauri/src/commands/agentic_one.rs`：`install_agent` 只安装 CLI，不再顺带执行 MCP 安装器。
- **新增手动 MCP 安装入口**：
  - `apps/desktop-tauri/src-tauri/src/commands/agentic_one.rs`：新增 `agent_mcp_install_status` 与 `install_agent_mcp`。
  - `apps/desktop-tauri/src-tauri/src/lib.rs`：已注册上述两个 Tauri command。
  - `tools/gto-agent-mcp/bin/gto-agent-mcp-install.mjs` / `tools/gto-agent-mcp/src/installer.mjs`：支持 `--target claude|codex|gemini|qwen` 定向安装。
- **设置页 agent 卡片支持独立 MCP 状态/按钮**：
  - `crates/vb-ai-config/src/models.rs` / `crates/vb-ai-config/src/service.rs`：`AiAgentSnapshotCard` 新增 `mcp_installed`，Claude/Codex/Gemini snapshot 都会回填。
  - `apps/desktop-web/src/features/settings/ai-providers/shared/ProviderAgentCard.tsx`：agent 卡片不再直接暴露 MCP，而是统一收口为“增强服务”入口。
  - `apps/desktop-web/src/features/settings/ai-providers/shared/AgentEnhancementsModal.tsx`：新增苹果风格独立弹窗，提供 `MCP / Skills` 两个 tab；当前 `MCP` tab 内展示 `GT Office Agent Bridge` 服务项。
  - `apps/desktop-web/src/features/settings/ai-providers/AiProvidersSection.tsx`：CLI 安装、配置入口与增强服务入口拆成三个独立动作。
  - `apps/desktop-web/src/features/settings/ai-providers/shared/provider-utils.ts`：新增增强服务状态归一化，明确区分 `not_installed / preconfigured / installed`，避免“CLI 未安装但仍显示 MCP 已安装”的误导。
  - `apps/desktop-web/src/shell/i18n/messages.ts`：相关文案改为“安装 CLI”与“安装 MCP”。
