# vbCode 上下文交接文档

## 当前里程碑与本轮目标

- 里程碑：M2 MVP 完整
- 任务：T-171
- 本轮目标：把设置页的 Agent 供应商能力从“安装卡片”升级为真正可用的 Agent 配置中心，重点完成 Claude 深配、Codex/Gemini 轻配，以及安全的预览/应用/审计闭环。

## 已完成

- **前端配置中心全面升级**：
  - 重构 `apps/desktop-web/src/features/settings/ai-providers/`，实现模块化 Agent 配置中心。
  - **Claude 深配**：5步向导、预设切换（DeepSeek/Kimi等）、API Key 托管、安全预览后应用。
  - **Codex/Gemini 轻配**：新增 API Key 设置，支持自动注入 `OPENAI_API_KEY` / `GOOGLE_API_KEY` 到终端环境。
  - **审计闭环**：实现 `AuditLogView` 组件，展现全量 Agent 配置变更历史（Mode, Provider, Key Changes）。
  - **UI/UX 优化**：遵循 Apple 风格设计令牌，统一 Stepper、Diff 预览与卡片交互。
- **后端能力增强**：
  - `vb-ai-config` 领域服务：支持 `read_snapshot`、`preview_patch`、`apply_patch` 通用化，涵盖 Claude/Codex/Gemini。
  - `vb-storage` 扩展：`SqliteAiConfigRepository` 支持 `query_audit_logs` 查询变更记录。
  - **环境变量注入**：`build_agent_runtime_env` 统一管理全量 Agent 的运行时环境，支持 `terminal.create` 自动加载。
  - **一键部署增强**：`install_agent` 在安装 CLI 后自动触发 `gto-agent-mcp-install.mjs` 脚本，实现 MCP 网桥一键配置。
- **类型安全与规范**：
  - 补齐 `desktop-api.ts` 的 `aiConfigListAuditLogs` 等新接口与联合类型。
  - 全链路通过 `tsc` 严格类型检查与 `cargo check` 工作区静态检查。
