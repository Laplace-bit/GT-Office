# GT Office 上下文交接文档

## 最近交接快照

- 里程碑：M2 MVP 完整
- 任务：T-172
- 本轮目标：把 channel 通讯中的 Codex agent 消息解析切到更准确的 session log 绑定，并避免后台扫描阻塞前端。

## 已完成

- **runtime / reply session 元数据贯通**：
  - `AgentRuntimeRegistration` 新增 `providerSession`。
  - `agent.runtime_register` / MCP bootstrap / external reply bind 已能透传 `providerSessionId/logPath/sessionStartedAtMs/discoveryConfidence`。
- **Codex session log 绑定增强**：
  - `vb-session-log` 支持显式 `logPath`、显式 `providerSessionId`、`cwd + bind 时间窗 + prompt anchor` 的分层发现。
  - 同 cwd 多个 rollout 共存时，不再只靠“最新 jsonl”猜测。
  - 增量读取新增 `delta` 输出，并处理半行 JSON carry 与 truncate 重置。
- **channel finalize 语义收口**：
  - `app_state` 会把发现到的 provider session 回写到 runtime。
  - Codex `session-log` finalize 改为只发送相对最近 preview 的新增尾段；若无新增，不重复向 channel 发整段正文。
- **前端不卡顿约束落地**：
  - external reply flush worker 中的 `session-log` 刷新改为 `spawn_blocking`，避免把文件扫描/读取压到前端交互线程。
- **消息来源可视调试**：
  - 外发到 channel 的正文末尾会临时追加 `[source=... confidence=... phase=...]`，便于只看消息内容就判断来源。
