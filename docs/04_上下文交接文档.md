# vbCode 上下文交接文档

## 当前里程碑与本轮目标

- 里程碑：M2 MVP 完整
- 任务：T-171
- 本轮目标：把 Claude 配置中心补成接近 `cc-switch` 的供应商管理体验，并补齐 saved provider 的编辑能力与持久化语义。

## 已完成

- **Claude 官方模式基线已补齐**：
  - 参考 `cc-switch`，`official` 模式现在会在 `preview_patch`、`read_snapshot` 和 `savedProviders[]` 中回填 Anthropic 官方基线信息，而不是把 `baseUrl/model/authScheme` 留空。
  - 当前官方基线为：`provider=Anthropic Official`、`baseUrl=https://api.anthropic.com`、`model=claude-sonnet-4-20250514`、`authScheme=ANTHROPIC_AUTH_TOKEN`。
  - 这组字段只用于配置展示、预览和记录；真正应用到系统级 `~/.claude/settings.json` 时，官方模式仍会清空 GT Office 托管的 Claude env 覆盖，回到原生 Claude CLI 托管模式。
- **Claude saved providers 已入库并可切换**：
  - `vb-storage` 的 `ai_config_saved_claude_providers` 继续按 workspace 保存 Claude 历史配置。
  - 每次 `ai_config.apply_patch` 成功后，当前 Claude 配置都会 upsert 到数据库，并标记为 active。
  - `ai_config_switch_saved_claude_provider` 已可直接切换当前生效项；旧的官方 saved provider 记录即使缺失 `baseUrl/model/authScheme`，现在也会在读取/切换时自动回填官方基线。
  - 已保存供应商列表现在按保存顺序稳定返回；从列表切换“当前配置”时只更新激活标记，不再因为 `is_active/last_applied` 变化把卡片重排到列表顶部。
- **Claude saved providers 已支持原位编辑**：
  - Claude saved provider 仍采用“自动生成 `savedProviderId` + workspace 内 fingerprint 去重”的持久化模型；相同配置不会生成重复记录。
  - `preview_patch` 现在支持从前端携带 `savedProviderId`，后端会先加载目标 saved provider，并在未重新输入 API Key 时复用该记录已有的托管密钥。
  - `apply_patch` 在编辑链路里会优先复用原 `savedProviderId` 更新同一条记录；只有 fingerprint 与其他记录完全一致时才允许折叠到已有记录，不会额外新增重复项。
  - Claude 已保存供应商列表新增“编辑配置”按钮，点击后直接复用 Claude 详情参数配置页进入 `preview -> apply -> audit` 流程。
- **Claude 配置入口结构已改为卡片子入口**：
  - Claude 弹窗内 `配置向导 / 已保存供应商` tab 已删除。
  - Agent 供应商列表中的 Claude 卡片现在提供两个独立子按钮：`配置向导`、`已保存供应商`。
  - 两个子按钮已改为水平两列布局。
- **Claude 预设详情页已支持自定义网关名称**：
  - 预设模式下的 `providerName` 输入框不再只读，用户可以直接修改显示名称。
  - 后端 preview/apply 现在会保留 preset 模式下传入的自定义 `providerName`，不再强制回落到预设内置名称。
  - 重新打开配置弹窗时，预设模式会优先回填用户保存过的自定义名称。
- **Claude saved provider 切换闪烁已收口**：
  - saved provider 切换现在直接使用 `ai_config_switch_saved_claude_provider` 返回的 `effective snapshot` 更新设置页，不再额外 `read_snapshot` 全量回拉。
  - saved provider 卡片已拆成 memo 化子组件，尽量只重绘激活态发生变化的条目。
  - saved 列表视图不再在切换成功后插入瞬时 success 文案，减少布局抖动。
- **Claude 详情页输入框裁剪已修复**：
  - 调整了详情页滚动内容区与表单 grid 的左侧留白，并给 input/select 补上宽度约束。
  - 靠左输入框左侧边框被裁剪的问题应已消失。
- **Claude 官方详情页展示已收口**：
  - 在向导里切到 `official` 时，详情页会直接展示官方 provider、endpoint、model 和 authScheme。
  - 预览页也会显示同一组官方基线，不再沿用上一个第三方供应商的残留值。
- **既有 system-level live settings 托管保持有效**：
  - GT Office 仍会同步受管 Claude env 到 `~/.claude/settings.json`，同时保留 `mcpServers`、`permissions` 和非受管 env。
  - Windows home 解析与已存在文件替换兜底仍在。
