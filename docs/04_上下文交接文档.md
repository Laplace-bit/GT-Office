# vbCode 上下文交接文档

## 当前里程碑与本轮目标

- 里程碑：M2 MVP 完整
- 任务：T-171
- 本轮目标：把 Claude 配置中心补成接近 `cc-switch` 的供应商管理体验，包括官方档案可切换、已保存供应商入库、列表查看与一键切换。

## 已完成

- **Claude 官方模式语义已修正**：
  - 选择 `official` 并应用后，不再只是“清空 GT Office 托管 env”，而是会保存一条官方 saved provider 档案，并把它设为当前生效配置。
  - 后续从 saved provider 列表切回官方时，会同步工作区配置与 `~/.claude/settings.json`，清除 GT Office 托管的 Claude env 覆盖，回到原生 Claude CLI 托管模式。
- **Claude saved providers 已入库**：
  - `vb-storage` 新增 `ai_config_saved_claude_providers` 表，按 workspace 保存 Claude 历史配置，字段覆盖 `mode/provider/baseUrl/model/auth/secretRef/active/lastApplied`。
  - 每次 `ai_config.apply_patch` 成功后，当前 Claude 配置都会 upsert 到这张表，并标记为 active。
  - snapshot 现在会把 `claude.savedProviders[]` 一并返回给前端。
- **切换命令已打通**：
  - 新增 Tauri command `ai_config_switch_saved_claude_provider`。
  - 切换时会读取数据库中的 saved provider，更新工作区 `.gtoffice/config.json`、同步 `~/.claude/settings.json`、切换 active saved provider，并写入审计日志。
  - 如果目标记录不存在，后端返回稳定错误码 `AI_CONFIG_SAVED_PROVIDER_NOT_FOUND`。
- **Claude 配置 UI 已新增列表页**：
  - Claude 弹窗顶部新增 `配置向导 / 已保存供应商` 两个页签。
  - “已保存供应商”页会显示历史 Claude 配置卡片，标记当前生效项，并支持一键切换。
  - 新文案已补到 `messages.ts`。
- **既有 system-level live settings 托管保持有效**：
  - GT Office 仍会同步受管 Claude env 到 `~/.claude/settings.json`，同时保留 `mcpServers`、`permissions` 和非受管 env。
  - Windows home 解析与已存在文件替换兜底仍在。
