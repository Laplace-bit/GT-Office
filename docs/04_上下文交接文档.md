6. 修正 rendered-screen 内部空行处理：空行不再把同一条回复切成多个 island，回复内部段落空行会在最终回传文本中保留。

8. 权限申请和工具执行改为整块静默：`Yes, allow ...`、`Esc to cancel · Tab to amend · ctrl+e to explain`、`● Bash(...)`、`● Read N file`、`Reading N file...`、`Shell cwd was reset ...`、`Worked for ...` 不再进入 channel 正文。
9. 当 terminal 正处于权限申请状态时，来自 channel 的 `1/2/3/yes/no/allow/deny` 短确认输入不再替换当前 reply session，最终答复仍归属于最初的问题消息。
10. `channel_adapter` 发送层新增长文本分片逻辑：preview 只发首片；finalize 先编辑/发送首片，再把剩余 continuation 分片按顺序补发，避免 Telegram 只收到前半段。
11. rendered-screen reply session 的 finalize 条件已收紧：有 snapshot 的会话必须满足“idle + ready prompt”才 finalize；仅 idle 但仍未回到 prompt 的长回答不会被提前结束。
12. 前端 `StationXtermTerminal` 已从 `xterm.buffer.active` 逆向扫描最近 prompt 锚点，不再只上传 viewport，可将“本轮 prompt 到当前 buffer 末尾”的整轮 turn 快照上报后端。
