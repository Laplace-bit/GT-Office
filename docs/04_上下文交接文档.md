### Decisions

1. Telegram 出站策略采用“节流编辑同一消息”而不是“逐 chunk 新消息”，以避免刷屏并保留流式体验。
2. 回传会话状态机沉到 `app_state`，`channel_adapter` 仅负责调度与 connector I/O，保持职责边界清晰。
3. polling offset 必须持久化，内存 offset 仅作为运行时缓存层。
4. 响应提取策略必须面向“通用 CLI 终端流”，不绑定特定模型/供应商标记。
5. 完成态判定优先依赖“稳定静默 + 预览已建立/会话结束”，避免任何单一 prompt 事件触发提前收敛。
