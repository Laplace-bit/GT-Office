## 已完成

- 已在 `AGENTS.md` 新增“后端模块化（MUST）”规则，约束 Tauri command 必须按前端 feature 对齐建目录、`app_state` 只负责全局装配、测试不得散落源码根目录等
- 已在 `AGENTS.md` 新增“前端模块化（MUST）”规则，核心约束包括：
  - `apps/desktop-web/src/features/` 是前端业务模块唯一落点
  - `shell/` 仅负责壳层编排、窗口框架、导航装配与平台集成
  - `components/` 仅放跨 feature 复用的纯展示/基础组件
  - 单一 feature 的 hooks/model/controller/style 必须优先内聚在 `features/<name>/`
  - `styles/` 仅放全局 tokens/foundations/utilities；feature 专属样式必须跟随 feature 组件
  - `stores/` 仅放跨 feature 全局状态；单一 feature 状态不得默认提升
  - 新增前端文件前必须先确认 feature 归属、通用性与现有目录复用可能
- 已在 `AGENTS.md` 的 Frontend 规范中新增：
  - 单文件或单目录持续膨胀时，优先拆分 feature 子模块，禁止默认继续向 `shell/`、全局 `components/`、全局 `stores/` 追加业务实现
- 已在 `AGENTS.md` 的 DoD 中改为：
  - 前后端入口层目录都必须与前端 feature 边界对齐
- 已同步更新：
  - `docs/03_项目开发进度跟踪.md`
  - `docs/04_上下文交接文档.md`
