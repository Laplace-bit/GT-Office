# vbCode 上下文交接文档

## 当前里程碑与本轮目标

- 里程碑：M2 MVP 完整
- 任务：T-171 / T-172
- 本轮目标：修复 Windows 安装包运行时反复弹出系统命令行窗口的问题，并确保该修复不影响 Linux/macOS 既有行为

## 已完成

- 已定位根因：生产包主进程是 `windows` 子系统，启动后没有附着控制台；后台拉起 `curl.exe`、`cmd.exe`、CLI 可执行文件时，Windows 会为这些控制台子进程分配新窗口。Telegram polling worker 每 `1500ms` 触发一次 `curl`，因此表现为“关掉窗口又重新弹出”
- 已新增基础设施模块 `apps/desktop-tauri/src-tauri/src/process_utils.rs`，统一为 Windows 后台子进程注入 `CREATE_NO_WINDOW`
- 已把该封装接入这些路径：
  - `apps/desktop-tauri/src-tauri/src/connectors/telegram/api.rs`
  - `apps/desktop-tauri/src-tauri/src/connectors/feishu/api.rs`
  - `apps/desktop-tauri/src-tauri/src/commands/tool_adapter/mod.rs`
  - `apps/desktop-tauri/src-tauri/src/commands/agentic_one.rs`
  - `crates/vb-tools/src/agent_installer.rs`
- 非 Windows 路径保持 no-op，不改变 Linux/macOS 现有子进程行为
- 文档已同步：
  - `docs/03_项目开发进度跟踪.md`
  - `docs/04_上下文交接文档.md`
  - `docs/05_高质量功能设计_核心工作流.md`
