# 快速验证指南 - Select下拉框深色模式修复

## 🎯 5分钟快速验证

### 第1步：启动应用（在WSL中）

```bash
cd /mnt/c/project/vbCode/apps/desktop-web
bash run-wsl.sh
```

或者在Windows PowerShell中：
```powershell
cd c:\project\vbCode\apps\desktop-web
.\start-wsl.bat
```

### 第2步：打开浏览器

访问: http://localhost:5173

### 第3步：打开Settings

点击右上角的 **"Open Settings"** 按钮（或"打开设置"）

### 第4步：切换到深色主题

在Settings对话框中：
1. 找到 **"Theme"** 下拉框
2. 当前可能是 "Light"
3. **点击下拉框** - 这是关键测试点！

### 第5步：验证下拉选项

当下拉框展开时，检查：

#### ✅ 应该看到（正确）：
- 📱 下拉菜单背景：**深灰色**（~RGB 72,72,74）
- 📝 选项文字：**白色** 或 **浅色**
- 📌 选项清晰可读
- 🎨 hover时有轻微高亮

#### ❌ 不应该看到（错误）：
- ⛔ 白色背景白色文字（无法阅读）
- ⛔ 黑色背景黑色文字（无法阅读）
- ⛔ 模糊或低对比度的选项

### 第6步：选择"Graphite Dark"

点击选择 "Graphite Dark" 主题

### 第7步：测试其他3个下拉框

现在主题已经是深色了，依次测试：

1. **Language** (语言) - 点击展开
   - ✅ 深色背景
   - ✅ 白色文字
   - ✅ 选项清晰可读

2. **UI Font** (界面字体) - 点击展开
   - ✅ 深色背景
   - ✅ 白色文字
   - ✅ 选项清晰可读

3. **Mono Font** (代码字体) - 点击展开
   - ✅ 深色背景
   - ✅ 白色文字
   - ✅ 选项清晰可读

### 第8步：切换回浅色主题验证

1. 在Theme下拉框中选择 "Light"
2. 再次点击所有下拉框
3. 确认在浅色模式下也正常：
   - ✅ 浅色背景
   - ✅ 深色文字
   - ✅ 选项清晰可读

## 🐛 如果仍然有问题

### 问题1: 下拉框仍然是白底白字

**可能原因**:
1. 浏览器缓存未清除
2. CSS未正确加载

**解决方案**:
```
1. 硬刷新浏览器: Ctrl + Shift + R (Windows/Linux) 或 Cmd + Shift + R (Mac)
2. 清除浏览器缓存
3. 重启开发服务器
```

### 问题2: 下拉箭头不显示

**可能原因**: CSS的SVG data URI未正确加载

**解决方案**:
```
1. 检查浏览器控制台是否有CSS错误
2. 确认index.css文件正确保存
3. 重启开发服务器
```

### 问题3: 浅色主题下拉框也是深色

**可能原因**: color-scheme条件选择器有问题

**检查**:
```
1. 打开浏览器开发者工具 (F12)
2. 检查 <html> 元素的 data-theme 属性
3. 应该是: data-theme="graphite-dark" (深色) 或 无此属性 (浅色)
```
